## Краткая теория (вводная часть)

### Как фронтенд общается с бэкендом

До этой лабораторной мы работали с API через curl и браузер. Теперь сделаем так, чтобы наша собственная HTML-страница обращалась к серверу и отображала данные.

Всё это происходит без перезагрузки страницы — пользователь добавляет мем, карточка появляется сразу

### Почему браузер блокирует запросы — CORS

Есть важный момент: браузер не разрешает JavaScript обращаться к серверу на другом порту. Наша страница открыта на порту 5500 (Live Server в VS Code), а API работает на порту 5000. Это разные адреса с точки зрения браузера — он заблокирует запрос и покажет ошибку в консоли.

**CORS (Cross-Origin Resource Sharing)** — это механизм безопасности браузера. Исправляется на стороне сервера: нужно явно разрешить запросы с нашего фронтенда. Мы сделаем это в Program.cs на Шаге 2.

### Что такое fetch?

**fetch** — встроенная функция JavaScript для отправки HTTP-запросов.

**fetch** — асинхронная операция: ответ от сервера приходит не сразу. Первый await ждёт прихода ответа (заголовков), второй await response.json() ждёт загрузки тела ответа. Без await вы получите объект Promise, а не реальные данные.

| Концепция                           | Описание                                |
| ----------------------------------- | --------------------------------------- |
| fetch(url)                          | GET-запрос — получить данные с сервера  |
| fetch(url, { method: 'POST', ... }) | POST-запрос — отправить данные          |
| fetch(url, { method: 'DELETE' })    | DELETE-запрос — удалить данные          |

| response.json()                     | Прочитать JSON из тела ответа           |

| async / await                       | Дождаться ответа от сервера             |
| try / catch / finally               | Обработка ошибок, гарантированный блок  |
